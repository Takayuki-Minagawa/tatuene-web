# 逹エネ断熱シミュレーター Web版 — 作業計画

## 背景・目的

`data/Web版_逹エネ断熱シミュレーターVer1-0-0.xlsx` は「逹エネ断熱シミュレーター（木造住宅ひと部屋断熱改修用）」。
このExcelには**作成者との契約と東京都の評価**が紐づいており、**計算ロジック（数式）を勝手に作り直すことは禁止**。
一方でExcel UIは一般ユーザーには扱いにくい。そこで、

> **計算はExcel原本の数式をそのまま動かし（再実装しない）、入力UIだけを使いやすいフォームに置き換える**

Webアプリを作る。これを自社「辰の達人診断」のユーザー様向けWebサイトとして提供する。
保存/読込はJSON、入力チェック後にPDF帳票（評価シート）をダウンロードできるようにする。

### 決定事項（確認済み）
- PDF帳票：**Excel（評価シート）を忠実に再現**（東京都評価・契約との整合を最優先）
- フレームワーク：**Next.js 静的書き出し（`output: export`）** + TypeScript + Tailwind
- 建材マスタ（部材性能シート98件）：**一般ユーザーも編集可能**（Excel同様、編集内容はJSONに含める）
- Excel(.xlsx)書き出し：**当面不要**（JSON保存/読込 + PDFのみ）

## Excel解析サマリ（設計の根拠）

| シート | 役割 | 行×列 | 備考 |
|---|---|---|---|
| 表紙 | 物件メタ情報入力＋表紙 | 59×12 | 工事名/住所/場所/建築年/診断者 等 |
| 計算シート（現状） | 主入力（寸法・開口部・建材） | 287×19 | 白セルのみ入力。建材はドロップダウン |
| 計算シート（改修後） | 改修部位・建材・面積 | 331×18 | 現状からの差分入力 |
| 評価シート | **診断帳票（PDF本体）** | 128×92 | 横向きA4。断熱等級/UA値/CO2・電気代削減/保温時間/工事費参考 |
| 部材性能シート | 建材マスタ（VLOOKUP元） | 171×9 | B12:B142（98件）。λ+厚み or U値からR値算出 |

- **数式総数 885。使用関数は `IF / AND / VLOOKUP / SUM / LEFT / VALUE / MID / LEN / ROUND` の9種のみ**。
  マクロ無し（.xlsx）、配列数式無し、揮発関数無し。→ **HyperFormula（JS計算エンジン）で完全に再現可能**。
- ドロップダウン：建材選択 = `部材性能シート!$B$12:$B$142` を全建材セルに適用。
- 既存の `#REF!` が12箇所（原本由来）。**原本どおり保持**（修正しない＝同一性維持）。
- 画像：ロゴ等のPNG/JPEGは流用可。EMF（Excelのグラフ）はWeb用に再現。

## アーキテクチャ（核心）

```
┌─────────────────────────────────────────────────────────────┐
│  ブラウザ（静的アプリ・サーバー不要）                        │
│                                                              │
│  [入力フォームUI] ──書込──▶ [HyperFormula エンジン] ◀──読出── │
│   表紙/現状/改修後/マスタ      (Excel原本の数式そのまま)        │
│        │                          │                          │
│        │                          ▼                          │
│        │                   [評価シート帳票] ──▶ PDFダウンロード│
│        ▼                                                      │
│   [JSON保存/読込] ◀── 入力値のみ（軽量・人間可読）            │
└─────────────────────────────────────────────────────────────┘
```

### 同一性の担保（最重要）
1. **ビルド時にExcel原本から「ワークブックモデルJSON」を抽出**（全セルの数式/値をアドレス→式で保持）。openpyxl で抽出するスクリプトを用意。
2. 実行時、そのJSONから **HyperFormula** インスタンスを構築。数式は原本の文字列をそのまま投入＝**一切書き換えない**。
3. フォームは「名前付き入力」を**特定セル**（例：`計算シート（現状）!D11`=床面積）に読み書きするだけ。
4. **忠実性テスト**：openpyxlの `data_only=True` で取得したExcelのキャッシュ計算値と、同一入力をHyperFormulaに与えた結果を突合し一致を検証（テストで自動化）。これで「下手に作り直していない」ことを機械的に証明。

### 保存JSON形式（例）
```json
{
  "app": "tatuene-insulation",
  "version": "1.0.0",
  "savedAt": "2026-05-31T12:00:00+09:00",
  "inputs": {
    "表紙!E30": "サンプル",
    "計算シート（現状）!D11": 9.93,
    "計算シート（現状）!D69": "窯業系サイディング ｔ15"
  },
  "masterOverrides": { "部材性能シート!F60": 0.038 }
}
```
読込＝`inputs` を順にエンジンへ `setCellContents` → 再計算 → フォーム反映。

## 入力チェック（データチェック）仕様

PDF出力前に実行。Excel注記（シート内の「◇記入上の注意」）をルール化：
- 必須：表紙の工事名/住所/建築年、現状の床面積・寸法。
- 数値域：面積・長さ・高さは正数。温度差・稼働時間は妥当域。
- 整合：**建材を選んだのに当該部位の面積が空 → 警告**（注記「面積を記入しないと計算に反映されない」）。
- 建材値：ドロップダウン候補（マスタB列）に存在する値のみ。空欄は「無し」。
- `#REF!`/`#N/A` など計算エラーセルの検出 → 該当入力を指摘。

結果は「エラー（出力不可）／警告（確認の上出力可）」に分類してパネル表示。

## PDF帳票（評価シートの忠実再現）

- 評価シートを**横向きA4のHTMLテンプレート**として再構成（CSS Grid/絶対配置でExcelレイアウトに一致）。値はHyperFormulaの計算結果をバインド。
- 断熱等級バー（等級1〜7）、UA値、各部熱損失、CO2/電気代/ガソリン/森林面積、保温時間、工事費参考（ケース1〜3）を配置。
- 「熱はどこから逃げている？」の円グラフ等（原本はEMF）は **SVG/Chart.js でライブ再現**。ロゴ等PNGは流用。
- 出力：印刷用CSS + ブラウザ印刷、かつ「PDFダウンロード」ボタン（html2canvas + jsPDF、または @react-pdf/renderer をレイアウト忠実度で選定）。

## 作業フェーズ

1. **足場づくり**：Next.js(静的) + TS + Tailwind 雛形。`extract-workbook.py` で workbook-model.json と画像を生成。
2. **計算エンジン**：HyperFormula 初期化ラッパ + cell-map。**忠実性テスト**を先に通す（同一性の証明＝最優先）。
3. **入力フォームUI（Excel風）**：表紙→計算(現状)→計算(改修後)→部材マスタ の順。タブ＋グリッド風レイアウト、建材ドロップダウン、リアルタイム再計算表示。
4. **保存/読込（JSON）**：エクスポート/インポート、入力反映。
5. **データチェック**：上記仕様の検証パネル。
6. **評価シート帳票 + PDF出力**：忠実レイアウト + チャート + ダウンロード。
7. **仕上げ/公開**：レスポンシブ調整、静的書き出し、ホスティング手順。

## レスポンシブ対応方針

- **PC/タブレット最適化＋スマホでも一応使える**を狙う（スマホ利用は稀のため過剰実装はしない）。
- ヘッダー・タブ・ボタンは折り返し＋タッチ可能サイズ。入力欄/ドロップダウンもタッチ操作に配慮。
- Excel風の幅広グリッド（開口部・建材表）は、スマホでは**横スクロール可能な表**として表示（縦積みに崩さずExcelとのセル対応を保つ）。
- PDF帳票（評価シート）は仕様上**横向きA4固定**。スマホは縮小プレビュー＋ピンチ拡大、ダウンロードPDFは全端末同一の正式レイアウト。

## 検証方法（完了確認）

- `python scripts/fidelity-check.py`：サンプル入力でExcel計算値とJS結果が**全セル一致**（許容誤差ほぼ0）。
- `npm test`（tests/fidelity.test.ts）：代表3〜5パターンの入力で総熱損失量・UA値・断熱等級・CO2削減が一致。
- 手動：`npm run dev` →（a）原本Excelと同じ入力を投入し評価シート数値が一致、（b）JSON保存→再読込で復元、（c）不備入力でチェックが警告、（d）PDFダウンロードがExcel印刷と同等レイアウト。
- `npm run build`（`output: export`）で `out/` が生成され、静的ホスティングで動作。
