## 2. 正確性・バグ

- **[高] ドラフト復元失敗時に壊れたドラフトが残り無限ループ化**
  - 該当: `src/app/page.tsx`（起動時の復元 effect）
  - 問題: 読込・初期化・保存では `clearDraft()`/`skipNextAutosave()` を呼ぶが、起動時のドラフト復元では呼ばない。`applyData` が例外を投げる（壊れた drawings 等）と catch でメッセージを出すだけで破棄しないため、次回起動でも同じドラフトで失敗し続ける。
  - 推奨: 復元の成功・失敗いずれでも適切に `clearDraft()`（最低でも失敗時は破棄してループを断つ）。

- **[中] 入力欄の先頭 `=` が数式評価され、保存ラウンドトリップで原文が失われる**
  - 該当: `src/engine/workbook.ts`（`coerce`）＋ `CellInput` / `collectInputs`
  - 問題: テキスト入力欄に `=1+1` 等を入れると HyperFormula が数式として評価。`getInputRaw` は評価結果を返すため、再表示・保存で原文 `=1+1` が `2` に化け、データが非可逆に失われる。エラー値（`#DIV/0!` 等）やオブジェクトが `String(raw)` で `[object Object]` 化する経路も同根。
  - 推奨: 入力欄（locked=False の非数式セル）では `=` 始まりをテキスト扱いにする。`getInputRaw` で非プリミティブ/エラーを正規化。

- **[中] 入力モード（行絞り込み）で結合セルが分割表示・重複してレイアウトが崩れる**
  - 該当: `src/components/SheetGrid.tsx`（`useInputOnly` 時に `anchors` と `covered` をともに無視）
  - 問題: 直近追加した入力モードでは結合を完全に無効化しているため、結合範囲に値があると各セルが個別表示され見出しが割れる。
  - 推奨: `covered` セルは隠したまま `anchors` の rowSpan/colSpan のみ 1 に落とす、または1セルに集約。

- **[中] `applyInputs` / `resetToDefaults` がセル単位の `setCellContents` を多数回呼び、毎回全再計算（読込・初期化が重い）** ※性能観点とも重複
  - 該当: `src/engine/workbook.ts`
  - 問題: 全入力欄を1つずつクリア→再設定（2周）し、その都度 HyperFormula が再評価。評価シート含め2,000回以上の設定で体感停止しうる。
  - 推奨: `hf.batch()` / `setSheetContent` でまとめて適用し再評価を1回に、`emit` も末尾1回に集約。

- **[中] PDF生成で巨大シートが canvas 寸法上限を超えても検出されず、欠損PDFを出力**
  - 該当: `src/lib/pdf.ts` / `src/components/ReportFrame.tsx`
  - 問題: `scale:2` × `width:max-content` の大表でブラウザの最大 canvas（各社 約16k〜32kpx）を超えると `toDataURL` が空/黒画像を返し、エラーにならず欠損PDFになる。
  - 推奨: canvas 寸法を検証し上限超過時はスケールを落とす／空 dataURL を検出して警告。

- **[低] `jumpNextEmpty` のインデックスが空欄数の変化でズレ、巡回が乱れる**
  - 該当: `src/app/page.tsx`
  - 問題: `jumpIdx` を `nodes.length` で剰余するため、入力で空欄が減るとDOM順と対応がずれ、次の空欄を飛ばすことがある。シート切替でもリセットされない。
  - 推奨: 「現在フォーカスより後の最初の `[data-empty]`」を都度探索する方式へ。

- **[低] 図面 transform の scale が無防備で、改竄/旧形式JSONの極端値がそのまま復元される**
  - 該当: `src/drawings/store.ts`（`setTransform` / `restore`）
  - 問題: ジェスチャ経由はクランプするが、ストア直接設定・復元はクランプなし。巨大/ゼロ scale で描画破綻。
  - 推奨: `restore`/`setTransform` で `[MIN_SCALE, MAX_SCALE]` にクランプ。

- **[低] 壊れた .zip/.json 読込で英語の生例外がトーストに出る**
  - 該当: `src/lib/storage.ts`（`JSON.parse` 等）
  - 推奨: parse を try/catch し「ファイルが壊れています」等の日本語メッセージへ正規化。

## 3. セキュリティ・プライバシー

> 前提: サーバ/DBがなく攻撃面は限定的。`dangerouslySetInnerHTML`/`eval` なし、シート名検証ありでプロトタイプ汚染耐性あり、秘密情報・APIキーの混入なし（grep確認済み）。実務上の主眼は「ローカル残留する個人情報」と「信頼できない保存ファイルの入力検証」。

- **[中] autosave / 保存ファイルに施主名・物件名・住所等が平文でローカル保存される**
  - 該当: `src/lib/autosave.ts`（localStorage `tatuene-draft:v1` / IndexedDB `tatuene-draft`）, `src/lib/storage.ts`（`buildSaveFile`）
  - リスク: 共有PC・キオスク端末で利用後に明示クリアしないと、後続利用者が devtools / localStorage から閲覧可能。
  - 推奨: ①ドラフト破棄導線の明示 ②「共有端末では個人情報が残る」旨の注意表示 ③一定時間でのドラフト失効。

- **[中] 保存ファイルの `imageType` を未検証で dataURL の MIME / ZIP内拡張子に流用**
  - 該当: `src/lib/storage.ts`, `src/drawings/store.ts`
  - リスク: 悪意ある保存ファイルで任意の `imageType` を注入可能。SVG `<image>` はスクリプト実行しないため XSS には至らないが、入力検証として不健全。
  - 推奨: `image/png|jpeg|gif|webp` の許可リストで検証し、非該当は `image/png` にフォールバック。`imageFile`/拡張子もサニタイズ。

- **[中] esbuild の高深刻度脆弱性（dev依存のみ・本番バンドルには非混入）**
  - 該当: `tsx`→esbuild, `vitest`→vite→esbuild（`npm audit` high 1件）
  - リスク: 開発者ローカルの dev サーバに限定。エンドユーザー影響なし。
  - 推奨: `npm audit fix`（緊急度低）。

- **[低] ZIP展開時の解凍量・エントリ数の上限なし（zip bomb / 自己被害DoS）**
  - 該当: `src/lib/storage.ts`（`JSZip.loadAsync`）
  - 推奨: 展開サイズ・エントリ数の上限ガードを追加すると堅牢。

- **[低] パスワードゲートはアクセス制御として機能しない（仕様承知済み）**
  - 該当: `src/components/PasswordGate.tsx`（`PASSWORD = "ebi-ken"` が公開ソース/バンドルに平文）
  - 推奨（テスト用途の最低限）: 本当に非公開が必要ならリポジトリを private 化、または Cloudflare Access 等ホスティング側認証へ。簡易ハッシュ比較で「チラ見」抑止を少し上げる程度は可能。

## 4. 設計・保守性

  - 該当: `.github/workflows/deploy.yml`

  - 推奨: build の前段に lint / `tsc --noEmit` / `npm run test` を実行する job を追加し、失敗時はデプロイをブロック。**最小工数で効果最大。**

## 5. パフォーマンス

> 実測: 最大チャンク **1.12MB**（HyperFormula＋574KB JSON）。jspdf/html2canvas/jszip は動的importで分離済み（良好）。評価シート **11,776セル**、計算シート（改修後）約6,000セル/102ドロップダウン（各98 option）。メモ化は実質ゼロ。

- **[高] 入力1回ごとにアクティブシート全体＋常駐ReportFrameが丸ごと再描画**
  - 該当: `src/engine/store.ts`（`emit`→全listener）, `src/components/SheetGrid.tsx`（`useEngineVersion()`）, 全 `CellInput`
  - 問題: `setInput`→version 更新で SheetGrid（メモ化なし・派生値全再計算・数千 `<td>` 再生成）と全 CellInput と ReportFrame が同時再描画。1キーストローク（blur）ごとに数十〜100ms級のジャンク要因。
  - 推奨: セル単位購読へ。数式セルと CellInput だけが購読し SheetGrid 本体は購読しない。最低でも `React.memo` ＋ 派生値の `useMemo`。理想は「変更セルIDセット」付き emit。

- **[高] 評価シート(ReportFrame)を起動時から常時オフスクリーン描画**
  - 該当: `src/app/page.tsx`, `src/components/ReportFrame.tsx`
  - 問題: PDF捕捉用の11,776セルが常駐DOM化し、入力ごとの再描画波及も受ける。PDFは押下時のみ必要。
  - 推奨: PDF生成時のみマウントする遅延描画に変更。**費用対効果が高く改修も局所的。**

- **[高] 建材ドロップダウンが大量の `<option>` を全数描画**
  - 該当: `src/components/CellInput.tsx`, `src/lib/materials.ts`
  - 問題: 1セル98 option × 102個 ≈ 約1万ノードを初期生成し、再描画のたびに作り直し。
  - 推奨: 不変 option リストをモジュール定数/メモでキャッシュ。`CellInput` を `React.memo` 化。

- **[中] 初期チャンク1.1MB（HyperFormula＋574KB JSON）**
  - 該当: `src/engine/workbook.ts`（`import modelJson`）
  - 推奨: JSONを `fetch` で動的取得＋スプラッシュ、描画専用メタと計算データの分離、HF初期化の遅延。

- **[中] `emptyCount` を毎レンダリングで全入力走査**
  - 該当: `src/app/page.tsx`
  - 推奨: `useMemo([active, version])` でキャッシュ。`dropdownSet` 生成もメモ化。理想は store 側で差分維持。

- **[低] `SheetGrid` の派生値（merges/colPx/rowPx/Set）をレンダー本体で毎回再構築**
  - 推奨: `useMemo([model, scale])` で固定。

## 6. アクセシビリティ・UX

- **[高] グリッド内のキーボードナビゲーションが事実上ない**
  - 該当: `src/components/CellInput.tsx`（Enterは `blur()` のみ）, `src/components/SheetGrid.tsx`
  - 課題: Excel風フォームなのに「Enterで下」「矢印で隣」「Tabで次の入力欄」が動かず、Enter後はフォーカスが消える。連続入力でマウス併用を強いられ、SR利用者は現在位置を見失う。
  - 推奨: `data-input-addr` を使い Enter/矢印で入力セル間移動。最低でも Enter で「次の空欄」へ送る。

- **[高] 入力欄に項目ラベル（aria-label）がなく、表のセマンティクスが弱い**
  - 該当: `src/components/SheetGrid.tsx`（全セル `<td>`、`th`/`scope`/`caption` なし）
  - 課題: SRでは各入力欄が「どの項目か」を読めず、項目名のないテキストボックスが延々と並ぶ。視覚に頼れない実務者が入力不能。
  - 推奨: 近傍ラベルセルの文字を `aria-label` として付与。表に `<caption>`（シート名）、見出しを `<th scope>` 化できればなお良い。

- **[高] 数値セルに `inputMode` 指定がなく、タブレットで数字キーパッドが出ない**
  - 該当: `src/components/CellInput.tsx`
  - 課題: 数値入力中心かつタブレット利用推奨なのに、現場での数値入力が遅く誤りやすい。
  - 推奨: 数値セルに `inputMode="decimal"`（または `"numeric"`）をモデル種別に応じて付与。

- **[中] 状態通知（保存/エラー/PDF生成中）がSRに届かない**
  - 該当: `src/app/page.tsx`（トーストに `role`/`aria-live` なし）
  - 推奨: トーストに `role="status" aria-live="polite"`（エラーは `assertive`）。空欄ジャンプ時は移動先項目名をライブリージョンで通知。

- **[中] CheckPanelの問題項目がキーボード操作不可（`<li onClick>`）**
  - 該当: `src/components/CheckPanel.tsx`
  - 推奨: 各項目を `<button>` 化し Enter/Space で `onJump` 発火。

- **[中] フォーカストラップが disabled/非表示要素を考慮せず、初期フォーカスが常に×ボタン**
  - 該当: `src/components/useModalA11y.ts`
  - 推奨: focusable 抽出に `:not([disabled])` ＋可視判定。初期フォーカスはダイアログ本体に当ててタイトルから読み上げ。

- **[中] コントラスト不足（空欄ハイライト・半透明バージョン入力欄）**
  - 該当: `globals.css`（空欄 `#fffbeb` と白背景の差が僅少＝色のみ依存、ヘッダー上の `rgba(255,255,255,0.15)` 入力欄）
  - 推奨: 空欄は背景差＋非色の手がかり（枠線/アイコン）を併用。バージョン入力欄を不透明＋AA基準（4.5:1）に。

- **[中] 絵文字依存ボタン＋狭幅でのツールバー過密**
  - 該当: `src/app/page.tsx`（「📂 読込」等、タブ行にタブ/入力モード/ズーム4種/初期化が横並び）
  - 推奨: 各ボタンに明示 `aria-label`、絵文字は `aria-hidden` 装飾化。狭幅ではズーム群のまとめ/オーバーフローメニュー化。

- **[低] ズーム群に `role="group"`/文脈付き通知がない**
  - 推奨: `role="group" aria-label="表示倍率"`、倍率変更時に「125% に拡大」のように通知。

## 7. 推奨ロードマップ（着手順の目安）

1. **守りを固める（低コスト・高効果）**
   - P1: CIに lint / `tsc --noEmit` / `npm run test`（忠実性含む）ゲートを追加
   - P5: ドラフト復元失敗時の `clearDraft()`
   - エンジン層テストの最初の一群（`coerce`・`formatByNumFmt`・保存往復）
2. **体感を上げる（性能）**
   - P2系: ReportFrame のオンデマンド化 → セル単位購読＋`React.memo`/`useMemo` → ドロップダウン option キャッシュ
3. **入力体験（UX/A11y）**
   - P3: グリッドのキーボード移動（Enter/矢印）→ 数値 `inputMode` → 入力欄 `aria-label`
   - トースト `aria-live`、CheckPanel のボタン化、コントラスト調整
4. **正確性の地ならし**
   - 入力欄 `=` のテキスト扱い、入力モードの結合表示修正、PDF canvas 上限の検出、`applyInputs` の `hf.batch` 化
5. **設計の整理（中期）**
   - `page.tsx` のフック分離、エンジン・シングルトン統一、外部JSONの shape 検証、ドキュメント追従
